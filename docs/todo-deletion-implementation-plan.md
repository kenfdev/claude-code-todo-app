# Todo削除機能実装計画

## 概要
ユーザーストーリー「Todo削除」の実装計画書です。現在のコードベースを分析した結果、基本的な削除機能は既に実装されていることが判明しました。本計画書では、現状の実装内容と改善提案をまとめます。

## ストーリー詳細
**As a** ログイン済みユーザー  
**I want to** 不要なTodoを削除したい  
**So that** リストを整理できる

## 受け入れ条件
- ✅ 削除ボタンでTodoを削除できる
- ✅ 削除前に確認メッセージが表示される
- ✅ 削除したTodoは即座に一覧から消える
- ✅ 削除操作のキャンセルも可能

## 現在の実装状況

### バックエンドAPI ✅
**エンドポイント**: `DELETE /api/todos/:id`
- **ファイル**: `app/routes/api.todos.$id.ts`
- **実装内容**:
  - 認証チェック（Bearerトークン必須）
  - 権限チェック（自分のTodoのみ削除可能）
  - 物理削除を実行
  - エラーハンドリング（404、500）

### フロントエンド ✅
**コンポーネント**: `TodoItem`
- **ファイル**: `app/components/todo-item.tsx`
- **実装内容**:
  - 削除ボタン（ゴミ箱アイコン）
  - `window.confirm`による確認ダイアログ
  - 削除中のローディング状態管理
  - エラーハンドリング（コンソールログ）
  - 完了済みTodoでは削除ボタンを非表示

### データベース層 ✅
**サービス**: `TodoService`
- **ファイル**: `app/services/todo.ts`
- **実装内容**:
  - `deleteTodo`メソッド
  - ユーザーID検証
  - 物理削除の実行
  - トランザクション処理なし（単一操作のため不要）

### テスト ✅
**E2Eテスト**: `todo-flow.spec.ts`
- **テストケース**: "should allow user to delete a todo"
- **カバー範囲**:
  - Todo作成
  - 確認ダイアログの表示と承認
  - 削除後の画面更新確認

## 改善提案

### 1. UX改善
現在の実装は基本的な要件を満たしていますが、以下の改善が可能です：

#### a. 確認ダイアログのカスタマイズ
**現状**: ブラウザネイティブの`window.confirm`を使用
```typescript
if (!window.confirm("このTodoを削除しますか？")) {
  return;
}
```

**改善案**: カスタムモーダルダイアログの実装
```typescript
// 新しいコンポーネント: DeleteConfirmationModal
interface DeleteConfirmationModalProps {
  isOpen: boolean;
  todoTitle: string;
  onConfirm: () => void;
  onCancel: () => void;
}

// モーダル内容:
// - Todo名を含む確認メッセージ
// - 削除の危険性を示す赤い警告アイコン
// - 「削除」ボタンは赤色で強調
// - ESCキーやオーバーレイクリックでキャンセル可能
```

#### b. 削除完了時のフィードバック
**現状**: 削除後、Todoが単に消える
**改善案**: 
- 削除成功時のトースト通知
- 「元に戻す」オプション（5秒間）

#### c. 一括削除機能
**改善案**:
- 完了済みTodo一括削除ボタン
- 複数選択モードと一括削除

### 2. エラーハンドリングの改善
**現状**: エラーはコンソールログのみ
```typescript
console.error("Todoの削除に失敗しました:", error);
```

**改善案**: ユーザーへのエラー通知
```typescript
// use-todos.ts に追加
const [deleteError, setDeleteError] = useState<string | null>(null);

// エラー時の処理
setDeleteError("削除に失敗しました。もう一度お試しください。");
```

### 3. アクセシビリティの改善
- 削除ボタンに`aria-label`追加
- キーボードショートカット（Delete/Backspaceキー）
- スクリーンリーダー用の削除完了通知

### 4. パフォーマンス最適化
**現状**: 削除後、全Todoリストを再取得
**改善案**: 楽観的更新
```typescript
// 即座にUIから削除
setTodos(prev => prev.filter(t => t.id !== todoId));

// バックグラウンドでAPIコール
// 失敗時のみロールバック
```

### 5. 監査ログ機能（オプション）
- 削除履歴の保存
- ソフトデリートオプション
- 削除済みTodoの復元機能
